上下文如何传递

任务间上下文传递有 3 层机制：

  1. 项目简报 — 持久化的 Markdown 文件，所有任务的 agent 启动时读取
  2. 上下文记录 — 结构化日志（insight/decision/warning/note），可关联到具体任务
  3. 任务结论 — 每个已完成任务的 conclusion + modified_files + notes 字段

Agent 执行任务时，autorun 的 prompt 会注入：
① 项目简报全文
② 相关任务（blocked_by / related）的 conclusion
③ 最近 10 条上下文记录
